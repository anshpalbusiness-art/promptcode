### Installation

```bash
# YOLO
curl -fsSL https://promptcode.ai/install | bash

# Package managers
npm i -g promptcode@latest        # or bun/pnpm/yarn
scoop install promptcode             # Windows
choco install promptcode             # Windows
brew install anshpalbusiness-art/tap/promptcode # macOS and Linux (recommended, always up to date)
brew install promptcode              # macOS and Linux (official brew formula, updated less)
sudo pacman -S promptcode            # Arch Linux (Stable)
paru -S promptcode-bin               # Arch Linux (Latest from AUR)
mise use -g promptcode               # Any OS
nix run nixpkgs#promptcode           # or github:anshpalbusiness-art/promptcode for latest dev branch
```

> [!TIP]
> Remove versions older than 0.1.x before installing.

### Desktop App (BETA)

PromptCode is also available as a desktop application. Download directly from the [releases page](https://github.com/anshpalbusiness-art/promptcode/releases) or [promptcode.ai/download](https://promptcode.ai/download).

| Platform              | Download                                |
| --------------------- | --------------------------------------- |
| macOS (Apple Silicon) | `promptcode-desktop-darwin-aarch64.dmg` |
| macOS (Intel)         | `promptcode-desktop-darwin-x64.dmg`     |
| Windows               | `promptcode-desktop-windows-x64.exe`    |
| Linux                 | `.deb`, `.rpm`, or AppImage             |

```bash
# macOS (Homebrew)
brew install --cask promptcode-desktop
# Windows (Scoop)
scoop bucket add extras; scoop install extras/promptcode-desktop
```

#### Installation Directory

The install script respects the following priority order for the installation path:

1. `$PROMPTCODE_INSTALL_DIR` - Custom installation directory
2. `$XDG_BIN_DIR` - XDG Base Directory Specification compliant path
3. `$HOME/bin` - Standard user binary directory (if it exists or can be created)
4. `$HOME/.promptcode/bin` - Default fallback

```bash
# Examples
PROMPTCODE_INSTALL_DIR=/usr/local/bin curl -fsSL https://promptcode.ai/install | bash
XDG_BIN_DIR=$HOME/.local/bin curl -fsSL https://promptcode.ai/install | bash
```

### Agents

PromptCode includes two built-in agents you can switch between with the `Tab` key.

- **build** - Default, full-access agent for development work
- **plan** - Read-only agent for analysis and code exploration
  - Denies file edits by default
  - Asks permission before running bash commands
  - Ideal for exploring unfamiliar codebases or planning changes

Also included is a **general** subagent for complex searches and multistep tasks.
This is used internally and can be invoked using `@general` in messages.

Learn more about [agents](https://promptcode.ai/docs/agents).

### FAQ

#### How is this different from Claude Code?

It's very similar to Claude Code in terms of capability. Here are the key differences:

- 100% open source
- Not coupled to any provider. Although we recommend the models we provide through [PromptCode Zen](https://promptcode.ai/zen), PromptCode can be used with Claude, OpenAI, Google, or even local models. As models evolve, the gaps between them will close and pricing will drop, so being provider-agnostic is important.
- Out-of-the-box LSP support
- A focus on TUI. PromptCode is built by neovim users and the creators of [terminal.shop](https://terminal.shop); we are going to push the limits of what's possible in the terminal.
- A client/server architecture. This, for example, can allow PromptCode to run on your computer while you drive it remotely from a mobile app, meaning that the TUI frontend is just one of the possible clients.
